1.用户排查

查看本地用户和组

win + r 输入 lusrmgr.msc 查看本地用户和组中是否有可疑用户

查看注册表

win + r 输入 regedit
HKLM\SAM\SAM\Domains\Account\Users\Names 查看是否有可疑用户
ps:记得赋予Administrator对应SAM的完全控制

利用Wmic命令

在cmd中输入：wmic useraccount get name,sid


特别关注带有$的隐藏用户

2. 启动项排查

启动菜单查看

C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

系统配置msconfig

win + r 输入 msconfig ---> 点击启动 查看可疑启动项

注册表

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce
HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce

3. 计划任务排查

利用计算机管理

win + r 输入 compmgmt.msc ---> 任务计划程序 ---> 任务计划程序库 查找可疑计划任务

命令行查看计划任务

cmd 中 输入：schtasks

4. 服务自启动排查

win + r 输入 services.msc    注意服务状态和启动类型，检查是否有异常服务

5. 临时目录排查

C:\Windows\Temp
%Temp%
在目录中发现可疑的pe文件即可丢入查杀平台进行验证，若是木马文件即需要即使取样并清除

6. 最近操作排查

win + r 输入 recent
时间排序，查看是否有可疑⽂件

7. 进程排查

查看可疑网络连接

netstat -ano | findstr "ESTABLISHED"

根据可疑连接pid定位

tasklist | findstr "pid"

杀掉进程

taskkill /F /T /PID [pid]
/F：强制终止
/T：终止该进程以及其子进程
/PID：后接进程pid值

利用msinfo32排查进程

win + r 输入 msinfo32 ---> 软件环境 ---> 正在运行任务 即可查看正在运行进程详细信息

8. 补丁排查

systeminfo 查看主机补丁情况，将其复制到在线Windows提权网站即可看到缺失补丁信息

9. 日志排查

查看日志

win + r 输入 eventvwr.msc 打开事件检查器
日志路径：C:\Windows\System32\winevt\Logs

重点排查事件

4624（登陆成功）4625（登陆失败）
关注调用进程名，登录源ip，登录源端口，登陆用户名和登陆失败信息等
4720（创建用户）4726（删除用户）
